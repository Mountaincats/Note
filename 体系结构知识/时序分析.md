# FPGA 时序分析核心笔记

## 一、基本结构
所有时序分析基于一条路径：
**Launch FF → 组合逻辑 → Capture FF**
- **起点（Startpoint）**：Launch FF.Q
- **终点（Endpoint / 路径点）**：Capture FF.D
- **N 个触发器 = N 个路径点**（每个 D 端都要检查）

## 二、三个核心约束

### 1. Setup Time（建立时间）
- **含义**：时钟沿**前**，数据必须提前稳定 `Tsu`
- **检查目标**：Capture FF.D
- **公式**
  \[
  T_{launch} + T_{co} + T_{data} \le T_{cycle} + T_{capture} - T_{su}
  \]
- **Slack**
  \[
  Slack_{setup} = T_{cycle} + T_{capture} - T_{su} - (T_{launch} + T_{co} + T_{data})
  \]
- **Slack > 0：满足**
- **Slack < 0：数据太慢，违例**
- **决定最高工作频率**

### 2. Hold Time（保持时间）
- **含义**：时钟沿**后**，数据必须继续稳定 `Th`
- **检查目标**：Capture FF.D
- **公式**
  \[
  T_{cycle} + T_{launch} + T_{co} + T_{data} \ge T_{capture} + T_{h}
  \]
- **Slack**
  \[
  Slack_{hold} = T_{cycle} + T_{launch} + T_{co} + T_{data} - (T_{capture} + T_{h})
  \]
- **Slack > 0：满足**
- **Slack < 0：数据太快，违例（极危险，降频无用）**

### 3. Pulse Width（脉冲宽度）
- **含义**：时钟高/低电平不能过窄
- **检查对象**：时钟本身，与数据路径无关
- 要求：`Thigh ≥ Twh_min`、`Tlow ≥ Twl_min`

## 三、关键延迟说明
- **Tco**：FF 内部时钟→输出延迟（必须计入）
- **Tsu / Th**：FF 内部建立/保持要求
- **Tdata**：组合逻辑延迟
- **Tlaunch / Tcapture**：时钟网络延迟(Clock Latency)，由 Vivado 布局布线后，根据实际时钟布线长度、RC 参数计算得出，即时钟从源点到对应触发器时钟端的真实延迟
- **Clock Skew**：时钟偏差
	\[
		Skew = Tcapture - Tlaunch
	\]
  - Skew 正：Capture 晚 → 帮 Setup
  - Skew 负：Capture 早 → 害 Setup
  - **全局时钟树BUFG**就是为了：**让所有 FF 的 Tlaunch ≈ Tcapture，Skew ≈ 0**

## 四、时序报告指标含义
- **WNS**：最差建立松弛（所有路径中 Setup Slack 最小值）
- **TNS**：总建立负松弛（所有 Setup 违例路径的负 Slack 之和）
- **WHS**：最差保持松弛（所有路径中 Hold Slack 最小值）
- **THS**：总保持负松弛（所有 Hold 违例路径的负 Slack 之和）
- **Failing Endpoints**：违例路径点数量（时序不满足的 Capture FF.D 端数量）
- **NA**：无任何违例，时序完全收敛（所有 Slack > 0）

## 五、核心结论
1. **所有时序分析只围绕 D 触发器**（FPGA 只有 DFF）
2. JK/SR/T 仅课本存在，实际由 DFF 模拟
3. Latch 电平敏感，FPGA 禁止使用
4. **Endpoint = 触发器 D 端**
5. Setup：沿前要稳；Hold：沿后要稳；Pulse Width：时钟别太窄
