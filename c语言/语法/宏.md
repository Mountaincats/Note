1. 使用gcc -E <name>.c -o <name>.i 生成预处理后文件
2. 展开过程：
   1. 先判断一个宏定义中有无## / # ，有则忽略参数中无其它宏，直接展开
   2. 若无## / # ，参数无其它宏，直接展开
   3. 若参数有其它宏，先展开，而该参数宏的展开过程与这三步相同
   4. 注：宏展开（包括参数宏）后得到的内容若是又组成了宏，则继续当成参数宏展开，可若新组成的宏（参数）包括了它的父级宏，则该参数宏不再展开如
   	``` bash
        \#define x y + z                                                             
        \#define y z + a
        \#define z x + a
        x
    ```
    x生成两个子宏y,z, 没有生成x这个宏，
    子宏y生成z,a, z再生成x, x是z这个子宏的父级宏，所以不再展开

3. 编译器内建特殊宏
    ```c
    __LINE__ 展开为当前代码行的行号，是一个整数
    __FILE__ 展开为当前源文件的文件名，是一个字符串
    __func__ 虽然不是宏，应该看作变量，不在预处理阶段展开。为当前运行函数的函数名的字符串(C99)
    ``` 

4. `#pragma预处理指示`供编译器实现一些非标准的特性，C标准没有规定#pragma后面应该写什么以及起什么作用，由编译器自己规定。有的编译器用#pragma定义一些特殊功能寄存器名，有的编译器用#pragma定位链接地址。如果编译器在代码中碰到不认识的#pragma指示则忽略它，例如gcc的#pragma指示都是`#pragma GCC ...`这种形式，用别的编译器编译则忽略这些指示。

5. `#if`后面的表达式:
	```
	#define VERSION  2
	#if defined x || y || VERSION < 3
	```
* 首先处理defined运算符，defined运算符一般用作表达式中的一部分，如果单独使用，#if defined x相当于#ifdef x，而#if !defined x相当于#ifndef x。在这个例子中，如果x这个宏有定义，则把defined x替换为1，否则替换为0，因此变成#if 0 || y || VERSION < 3。

* 然后把有定义的宏展开，变成#if 0 || y || 2 < 3。

* 把没有定义的宏替换成0，变成#if 0 || 0 || 2 < 3，注意，即使前面定义了一个变量名是y，在这一步也还是替换成0，因为#if的表达式必须在编译时求值，其中包含的名字只能是宏定义。

* 把得到的表达式0 || 0 || 2 < 3像C表达式一样求值，求值的结果是#if 1，因此条件成立。

6. `## 和 #`
* `##`运算符不仅限于函数式宏定义，变量式宏定义也可以用
* `#`只能用于函数式宏定义。`#`运算符后面应该跟一个形参自动用"号把**实参括起来成为一个字符串**，并且实参中的连续**多个空白字符被替换成一个空格**
