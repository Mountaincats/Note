1. 使用gcc -E <name>.c -o <name>.i 生成预处理后文件
2. 展开过程：宏展开发生在编译器预处理阶段，遵循**先参数，后替换，再扫描**的核心原则，并且只展开一次。

   1.  **参数分离与预扫描**
       *   当预处理器识别到一个宏名（例如 `MACRO(arg1, arg2)`），它首先**分离出参数**。
       *   接着，它会**预扫描每个参数**，检查其中是否包含 `#` 或 `##` 预处理运算符。这决定了下一步的行为。

   2.  **参数展开（条件性）**
       *   **如果宏的替换列表中包含 `#`（字符串化）或 `##`（记号连接）运算符**：则与该运算符**直接相连**的那个参数**不会被展开**，其文本被原样取出。其他参数按规则正常判断是否展开。
       *   **否则（替换列表中不含 `#` 或 `##`）**：则每个参数会被**独立地、完全地**展开。展开时，将参数本身视为一个独立的文本流，递归地应用本套展开规则。

   3.  **参数替换**
       *   经过步骤2处理（可能展开，也可能未展开）后的所有参数文本，被代入到宏的替换列表中，替换掉对应的形参。

   4.  **替换结果的重扫描与递归展开**
       *   完成参数替换后，生成的结果文本会与**之前的宏名**（即刚刚被展开的那个宏）一起，被放入一个“已展开宏”的集合中以防止循环。
       *   预处理器**重新扫描**这个结果文本。
           *   如果扫描到的记号是一个宏名，并且它**不在“已展开宏”集合中**，则对其递归地**从步骤1开始**整个展开过程。
           *   如果扫描到的记号是**本次展开的宏名自身**（即它的父级宏），则**停止对其展开**，以防止无限递归。这正是你例子中描述的情况。

   5. **防止无限递归**的规则，这是理解复杂宏展开的关键。

   	```c
   	#define x y + z
   	#define y z + a
   	#define z x + a
   	x
   	```
   	展开流程如下：
   	1.  遇到 `x`，开始展开。替换列表为 `y + z`，不含 `#` 或 `##`，所以先展开参数（这里 `x` 无参数）。
   	2.  替换：用 `y + z` 替换 `x`。结果文本变为 `y + z`。
   	3.  **重扫描**结果 `y + z`：
   		*   扫描到 `y`，`y` 不在“已展开集合”（目前只有 `x`）中，展开 `y`。
   			*   展开 `y` 得到 `z + a`，替换后重扫描。
   			*   扫描到 `z`，展开 `z`。
    				*   展开 `z` 得到 `x + a`，替换后重扫描。
    				*   **关键步骤**：重扫描 `x + a` 时，扫描到 `x`。此时 `x` 是正在展开 `z` 的“父级宏”（因为本次展开链是 `x -> y -> z`），`x` 已在“已展开集合”中。因此，**`x` 不再被展开**，保留为记号 `x`。
   			*   `z` 展开完毕，结果为 `x + a`。继续重扫描 `y` 展开后的剩余部分 `+ a`，无其他宏。最终 `y` 的展开结果为 `x + a + a`。
   		*   继续重扫描最初 `x` 展开结果 `y + z` 的剩余部分 `+ z`。
   			*   扫描到 `z`，展开 `z`，过程同上，结果同样为 `x + a`。
   	4.  整个 `x` 的最终展开结果为：`x + a + a + x + a`。

3. 编译器内建特殊宏
    ```c
    __LINE__ 展开为当前代码行的行号，是一个整数
    __FILE__ 展开为当前源文件的文件名，是一个字符串
    __func__ 虽然不是宏，应该看作变量，不在预处理阶段展开。为当前运行函数的函数名的字符串(C99)
    ``` 

4. `#pragma预处理指示`供编译器实现一些非标准的特性，C标准没有规定#pragma后面应该写什么以及起什么作用，由编译器自己规定。有的编译器用#pragma定义一些特殊功能寄存器名，有的编译器用#pragma定位链接地址。如果编译器在代码中碰到不认识的#pragma指示则忽略它，例如gcc的#pragma指示都是`#pragma GCC ...`这种形式，用别的编译器编译则忽略这些指示。

5. `#if`后面的表达式:
	```
	#define VERSION  2
	#if defined x || y || VERSION < 3
	```
* 首先处理defined运算符，defined运算符一般用作表达式中的一部分，如果单独使用，#if defined x相当于#ifdef x，而#if !defined x相当于#ifndef x。在这个例子中，如果x这个宏有定义，则把defined x替换为1，否则替换为0，因此变成#if 0 || y || VERSION < 3。

* 然后把有定义的宏展开，变成#if 0 || y || 2 < 3。

* 把没有定义的宏替换成0，变成#if 0 || 0 || 2 < 3，注意，即使前面定义了一个变量名是y，在这一步也还是替换成0，因为#if的表达式必须在编译时求值，其中包含的名字只能是宏定义。

* 把得到的表达式0 || 0 || 2 < 3像C表达式一样求值，求值的结果是#if 1，因此条件成立。

6. `## 和 #`
* `##`运算符不仅限于函数式宏定义，变量式宏定义也可以用
* `#`只能用于函数式宏定义。`#`运算符后面应该跟一个形参，自动用"号把**实参括起来成为一个字符串**，并且实参中的连续**多个空白字符被替换成一个空格**
