# 函数栈帧 + 递归空间复杂度

## 1. 栈帧空间大小决定因素
- 栈帧大小 = **函数进入时一次性分配**
- 分配大小在**编译期就确定**
- 大小 = **所有局部变量「同时存活」所需的最大空间** + 固定开销（返回地址、ebp等）

## 2. 变量分配时机
- **不是执行到声明才分配**
- 进函数时，栈空间**一次性分配完**
- 但变量的**生命周期/构造/初始化**是执行到定义时才开始

## 3. if/else 里的变量不会浪费空间
- 编译器做 **生命周期分析**
- **不同时存活的变量，栈空间复用**
- 栈帧只取 **所有分支中「最大需要空间」**，不会叠加
- 没进入的分支，变量**不占用额外空间**

## 4. 编译器计算栈帧大小的算法
1. 收集所有局部变量
2. 算大小、对齐
3. 分析**生命周期重叠**
4. 重叠->分开；不重叠 → 复用
5. 取**同时存活变量的最大总空间**
6. 加固定开销->最终栈帧大小

## 5. 递归空间复杂度 O(depth)
- 每层递归栈帧 = **固定大小 O(1)**
- 空间消耗 = **递归调用栈的最大深度**
- 所以空间复杂度 = **O(depth)**
