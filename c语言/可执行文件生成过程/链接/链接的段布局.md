下面是把你前面所有问题（linker script、`.text` 段、函数顺序、函数级 section 等）整理成一份**系统化笔记**，已经帮你按“从编译到运行”的逻辑串起来了，适合复习/面试/做 OS 实验用。

---

# 🧠 一、整体流程（最重要总览）

```text
C代码 → 编译 → .o目标文件 → 链接（linker script）→ ELF → 加载到内存运行
```

👉 每一步职责：

| 阶段   | 做什么                  |
| ---- | -------------------- |
| 编译器  | 把函数放进 `.text`        |
| 汇编器  | 生成 `.o`，包含各个 section |
| 链接器  | 合并 section + 决定内存布局  |
| 链接脚本 | 精确控制“放在哪里”           |

---

# 🧱 二、section 基础概念

## 常见段

| 段名        | 内容              |
| --------- | --------------- |
| `.text`   | 代码（函数）          |
| `.rodata` | 只读数据（字符串、const） |
| `.data`   | 已初始化全局变量        |
| `.bss`    | 未初始化全局变量（运行时清零） |

---

# 📦 三、.text 段是怎么来的？

## 1️⃣ 编译器决定“属于谁”

```c
void foo() {}
```

→ 编译后：

```asm
.section .text
foo:
```

👉 所有函数默认进 `.text`

---

## 2️⃣ 链接器决定“放哪里”

你的 linker script：

```ld
. = _pmem_start + _entry_offset;

.text : {
    *(entry)
    *(.text*)
}
```

👉 含义：

* 设置起始地址
* 收集所有 `.text` 相关内容
* 拼成一个大 `.text`

---

# 📍 四、内存布局是怎么确定的

## 核心机制：位置计数器 `.`

```ld
. = 0x80000000;
```

👉 表示：

```text
当前地址 = 0x80000000
```

---

## 按顺序排布

```ld
.text   → .rodata → .data → .bss → stack → heap
```

👉 **顺序完全由 linker script 决定**

---

## 📊 结合你的程序实际布局

```text
0x80000000  .text
            |_start
            |check
            |main
            |halt
            |_trm_init

0x80000380  .rodata
            |test_data

0x800003c0  .bss
            |_bss_start

            ↓
            stack (32KB)

0x80009000  _end (sp 初始值)
```

---

# 🚀 五、入口函数 `_start`

```ld
ENTRY(_start)
```

👉 告诉链接器：

```text
程序入口 = _start
```

---

同时：

```ld
*(entry)
```

👉 强制：

```text
entry 段放在最前面
```

---

# 🔢 六、函数在 .text 里的顺序（重点）

## 默认规则（非常重要）

> **顺序 = entry 段优先 + .o 文件顺序 + 源码顺序**

---

## 你的程序顺序

```text
_start      ← entry 段
check
main
halt
_trm_init
```

---

## 为什么是这个顺序？

### 1️⃣ `_start` 最前

```ld
*(entry)
```

---

### 2️⃣ 其他函数

👉 来自同一个 `.c` 文件：

```c
check → main → halt → _trm_init
```

👉 编译器按源码顺序生成

👉 链接器按顺序拼接

---

# ⚠️ 七、关键误区

| 误区                     | 正确         |
| ---------------------- | ---------- |
| `.text` 是 linker 创建的   | ❌ 编译器早就生成了 |
| 函数顺序由 linker script 控制 | ❌ 默认不是     |
| `*(.text*)` 会排序        | ❌ 只是收集     |

---

# 🧩 八、函数级 section（重点进阶）

## 什么是函数级 section？

编译选项：

```bash
-ffunction-sections
```

---

## 默认情况

```text
.text:
  foo
  bar
```

---

## 开启后

```text
.text.foo
.text.bar
```

👉 每个函数一个 section

---

## 作用：删除未使用函数

配合：

```bash
-Wl,--gc-sections
```

---

### 示例

```c
void used() {}
void unused() {}
```

👉 最终：

```text
只保留 used()
```

---

## ⚠️ 对函数顺序的影响

开启后：

```ld
*(.text*)
```

👉 收集：

```text
.text.foo
.text.bar
```

但：

```text
顺序不再保证！
```

---

类似有数据级section

# 🔍 九、全局变量是怎么定位的

你看到的：

```asm
auipc s6, 0
addi  s6, s6, 848  # 80000380
```

👉 等价：

```c
s6 = &test_data;
```

---

## 原理

👉 PC 相对寻址（位置无关代码）

---

# 🧵 十、栈和堆的布局

你的 linker script：

```ld
_stack_top = ALIGN(0x1000);
. = _stack_top + 0x8000;
_stack_pointer = .;
```

---

## 含义

```text
对齐 → 分配32KB → 栈顶 → 栈向下长
```

---

## 在代码中体现

```asm
auipc sp,0x9
addi  sp,sp,-4  # 0x80009000
```

👉 设置初始 SP
