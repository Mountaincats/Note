**动态链接器**是操作系统运行可执行文件时，在程序**启动时**（或函数被调用时）动态完成“链接”工作的**一个特殊程序**。它是静态链接的延伸，发生在程序运行时。

### 核心概念
*   **静态链接器**（`ld`）：是 `gcc` 调用的一部分，在**编译的最后阶段**运行，把 `main.o` 和库代码**物理合并**成一个完整的、自包含的可执行文件。
*   **动态链接器**（例如 Linux 上的 `/lib64/ld-linux-x86-64.so.2`）：是操作系统的一部分，在**程序启动时**运行，负责“连接”程序和它依赖的共享库。

### 它是如何工作的？

1.  **准备阶段（编译/链接时）**：
    ```bash
    gcc main.c -lm -o myprog
    ```
    *   当使用 `-lm` 链接动态库（`libm.so`）时，**静态链接器**不将数学函数的二进制代码复制到 `myprog` 中。
    *   它只在 `myprog` 的可执行文件中：
        *   记录一个清单：“本程序需要 `libm.so.6` 这个共享库。”
        *   将程序中所有调用 `sqrt` 等函数的地方，标记为“**待解析**”。
        *   指定由哪个动态链接器程序来负责这个工作（例如，在文件头中写 `INTERP` 段，指定 `/lib64/ld-linux-x86-64.so.2`）。

2.  **运行时阶段（程序启动时）**：
    当你执行 `./myprog` 时，内核首先加载 `myprop` 的代码，然后发现它需要一个“翻译官”（动态链接器），于是内核：
    *   加载并启动**动态链接器**（`ld-linux-x86-64.so.2` 本身）。
    *   将控制权交给动态链接器。

3.  **动态链接器的工作流程**：
    动态链接器接管后，开始执行以下任务，这被称为**动态链接**：
    *   **依赖查找**：根据 `myprog` 文件中的清单，去系统预设的路径（如 `/lib`, `/usr/lib`，或由 `LD_LIBRARY_PATH` 环境变量指定）中寻找 `libm.so.6`、`libc.so.6` 等所有需要的共享库文件。
    *   **库加载**：将这些共享库文件加载到内存中（如果还没被其他程序加载过，就新加载；如果已加载，就共享其内存映像）。
    *   **符号解析与重定位**：
        *   在内存中，找到 `libm.so.6` 里 `sqrt` 函数的实际内存地址。
        *   回到 `myprog` 的代码中，将所有调用 `sqrt` 的“待解析”位置，**修正**为这个真实的内存地址（这个过程叫**重定位**）。
    *   **控制权转移**：所有符号解析完成后，动态链接器将控制权跳转到 `myprog` 的 `main` 函数入口，程序**真正开始执行**。

4.  **延迟绑定（Lazy Binding，PLT）**：
    为了优化启动速度，现代系统常使用“延迟绑定”。即动态链接器在启动时**不立即解析所有函数地址**，而是：
    *   第一次调用 `sqrt` 时，会先跳转到一个叫做**过程链接表** 的小存根代码。
    *   PLT 代码会调用动态链接器，让它去解析 `sqrt` 的真实地址，并更新PLT。
    *   之后所有对 `sqrt` 的调用就直接跳转到真实地址了。这实现了“用时才链接”。

### 为何需要动态链接器？

1.  **节省磁盘和内存**：多个程序可共享磁盘上一个 `.so` 文件，在内存中共享同一份代码。
2.  **便于更新**：更新 `libm.so.6` 库文件后，所有依赖它的程序无需重新编译，下次启动时自动使用新库。
3.  **灵活性**：可通过 `LD_LIBRARY_PATH` 或 `LD_PRELOAD` 指定加载不同版本的库，用于调试或兼容。
4.  **插件系统**：程序可在运行时通过 `dlopen()` 动态加载 `.so` 文件作为插件，这正是由动态链接器的功能（`libdl`）提供的。

### 可视化流程
```
源代码 (main.c)  -> 编译 -> main.o
       |
       v
静态链接器 (ld)  --> 生成 --> 可执行文件 (myprog)
(处理 -lm)       (内含：我的代码 + 依赖清单 [libm.so.6])
                                          |
                                          程序启动
                                          |
                                          v
内核加载 myprog --> 发现需动态链接 --> 启动动态链接器 (ld-linux)
                                          |
                                          v
动态链接器工作：查找 libm.so.6 -> 加载到内存 -> 修正 myprog 中所有 sqrt 调用地址
                                          |
                                          v
                                        控制权交还 --> myprog.main() 开始执行
```

**一句话总结**：**动态链接器是运行时的“连接员”**。它在程序启动的最后一刻，在内存中将程序的“未完成部分”和共享库的“实现部分”连接起来，使程序得以运行。
