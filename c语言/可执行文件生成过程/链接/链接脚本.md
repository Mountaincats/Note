### 一、链接器脚本的核心目的
链接器脚本（Linker Script）是给 GNU 链接器 `ld` 用的配置文件，核心目的是**精确控制程序的内存布局**——解决「编译后的目标文件（.o）如何组合成最终可执行文件/库，以及这些内容在内存中如何分布」的问题。

#### 为什么需要链接器脚本？
普通应用程序（如 PC 上的 C 程序）可以用 `ld` 的默认脚本，无需手动编写；但在**嵌入式系统、内核开发、裸机编程**等场景中，必须自定义内存布局，原因包括：
1. 硬件内存有明确分区（如片内 Flash、片外 RAM、外设地址空间），代码/数据必须放在指定地址；
2. 需要区分只读/可写、已初始化/未初始化数据段，适配硬件的读写权限；
3. 必须手动定义栈、堆的位置和大小（裸机无操作系统自动分配）；
4. 标记内存段的边界（如 BSS 段起始/结束），方便代码中手动管理内存（如清零 BSS）。

简单说：链接器脚本是「告诉链接器如何把零散的目标文件段，按我们想要的地址、顺序、属性，拼接成最终程序」的“布局图纸”。

### 二、链接器脚本的核心语法
链接器脚本的语法围绕「符号定义、段（Section）、内存区域、程序头」四大核心展开，下面结合常用语法规则和示例讲解，所有语法均基于 GNU `ld` 标准。

#### 1. 基础语法规则
- **注释**：`/* 多行注释 */` 或 `// 单行注释`（部分版本支持）；
- **符号定义**：`符号名 = 表达式;`（如 `_stack_top = ALIGN(0x1000);`），符号可在代码中通过 `extern 类型 符号名;` 引用；
- **位置计数器**：`.` 是核心内置符号，表示「当前内存地址」，可通过赋值（`. = 0x80000000;`）或运算（`. += 0x1000;`）修改；
- **表达式**：支持加减乘除、位运算、对齐函数（`ALIGN(n)`）、取地址（`ADDR(段名)`）等，如 `. = ALIGN(0x1000);`（对齐到 4KB 边界）。

#### 2. 核心指令详解
| 指令/语法          | 作用                                                                 | 示例                          |
|---------------------|----------------------------------------------------------------------|-------------------------------|
| `ENTRY(符号)`       | 指定程序入口地址（替代默认的 `main`，裸机常用 `_start`）              | `ENTRY(_start)`               |
| `SECTIONS { ... }`  | 定义段的布局（脚本核心），所有段的排列、地址都在其中配置              | 见下文示例                    |
| `PHDRS { ... }`     | 定义程序头表（仅可执行文件/共享库），告诉操作系统如何加载内存段       | `PHDRS { text PT_LOAD; }`     |
| `MEMORY { ... }`    | 定义硬件的内存区域（如 Flash/RAM），指定地址范围和属性（可选）        | `MEMORY { RAM : ORIGIN=0x80000000, LENGTH=1M; }` |
| `段名 : { ... }`    | 定义一个输出段，合并目标文件的输入段                                 | `.text : { *( .text ) }`      |
| `*(输入段)`         | 匹配所有目标文件中的指定输入段（`*` 是通配符）                       | `*(.text*)`（匹配所有.text段）|
| `: 程序头名`        | 将输出段关联到 PHDRS 中定义的程序头（指定加载属性）                  | `.text : { ... } : text`      |
| `ALIGN(n)`          | 将位置计数器对齐到 `n` 的整数倍（常用 4/0x1000 对齐）                | `_stack_top = ALIGN(0x1000);` |

#### 3. 常用语法示例（结合实际场景）
##### （1）定义内存区域（MEMORY）
嵌入式场景中，先声明硬件的内存分区，再将段映射到对应区域：
```ld
/* 定义内存：Flash（只读，存放代码）、RAM（可写，存放数据/栈/堆） */
MEMORY {
  FLASH : ORIGIN = 0x08000000, LENGTH = 128K  /* STM32 片内 Flash 起始地址 */
  RAM   : ORIGIN = 0x20000000, LENGTH = 20K   /* STM32 片内 RAM 起始地址 */
}

SECTIONS {
  /* 代码段放到 Flash 中 */
  .text : {
    *(.text)
  } > FLASH  /* > FLASH 表示映射到 MEMORY 定义的 FLASH 区域 */
  
  /* 数据段放到 RAM 中 */
  .data : {
    *(.data)
  } > RAM AT> FLASH  /* AT> FLASH 表示数据段的初始值存放在 Flash，启动时拷贝到 RAM */
}
```

##### （2）段的合并与顺序
链接器会按脚本中写的顺序合并输入段，比如先放启动代码，再放普通代码：
```ld
SECTIONS {
  . = 0x8000000;  /* 起始地址设为 0x8000000 */
  
  /* 先放启动代码的 entry 段，再放普通 text 段 */
  .text : {
    startup.o (entry)  /* 只取 startup.o 中的 entry 段 */
    *(.text)           /* 其他所有目标文件的 text 段 */
  }
  
  /* 已初始化数据段紧跟 text 段 */
  .data : {
    _data_start = .;
    *(.data)
    _data_end = .;
  }
}
```

##### （3）符号的使用（代码中引用）
脚本中定义的符号可在 C/汇编代码中直接使用，比如清零 BSS 段：
```c
// C 代码中引用链接器脚本定义的符号
extern char _bss_start, _bss_end;

void clear_bss(void) {
  // 遍历 BSS 段，将所有未初始化数据清零
  char *p = &_bss_start;
  while (p < &_bss_end) {
    *p++ = 0;
  }
}
```

### 三、总结
#### 1. 核心目的
- 自定义程序在内存中的**地址、顺序、权限**，适配硬件内存布局（嵌入式/裸机核心需求）；
- 标记内存段边界（如 BSS 段、栈/堆起始），方便代码中手动管理内存；
- 控制可执行文件的加载属性（通过 PHDRS），适配操作系统的内存加载规则。

#### 2. 核心语法要点
- `SECTIONS` 是脚本的核心，用于定义段的布局，`.` 是位置计数器；
- 符号定义（`符号 = 表达式;`）是代码与脚本交互的桥梁；
- `MEMORY`/`PHDRS` 是进阶语法，分别用于定义硬件内存分区、程序加载属性；
- 通配符 `*` 用于匹配多个目标文件的输入段，`ALIGN()` 用于内存地址对齐。

链接器脚本的本质是「用声明式语法描述内存布局」，掌握核心语法后，可根据硬件需求灵活调整代码、数据、栈、堆的位置，这也是嵌入式裸机开发的必备技能。
