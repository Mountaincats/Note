## 什么情况下会出现 C 和 C++ 混合编程？

### 常见场景(性能、底层、第三方库)：
1. **遗留系统维护**：核心代码库是 C 写的，但新功能用 C++ 开发
2. **性能敏感模块**：某些关键性能模块用 C 写，上层逻辑用 C++
3. **跨平台兼容**：某些硬件驱动或底层系统接口只提供 C API
4. **第三方库使用**：
   - 数据库（SQLite、PostgreSQL 客户端）
   - 图形库（OpenGL、Vulkan）
   - 科学计算（FFTW、BLAS/LAPACK）
   - 网络库（libcurl、OpenSSL）
5. **嵌入式系统**：硬件厂商提供 C 的 SDK
6. **操作系统内核**：内核接口通常是 C API

### 1. **语言互操作基础**
- `extern "C"` 链接规范的使用
- 名称修饰（name mangling）的差异

- 基本类型对应关系（`int`、`char` 等）

- `malloc`/`free` 与 `new`/`delete` 的混用
- 谁分配谁释放的原则
- 对齐和填充问题

### 4. **错误处理**
- C 的错误码与 C++ 异常的交互
- 资源泄漏预防
- 析构时的清理

### 5. **构建系统**
- 分别编译 C 和 C++ 源文件
- 链接器配置
- 头文件包含顺序

## 要注意的问题

### 1. **名称修饰问题**
```cpp
// C++ 文件
extern "C" {
    #include "c_lib.h"  // 确保 C 函数保持 C 名称
}

// 或者头文件中
#ifdef __cplusplus
extern "C" {
#endif
// 函数声明
#ifdef __cplusplus
}
#endif
```

### 2. **类型安全**
```cpp
// 危险：C 的 void* 隐式转换
void* ptr = malloc(100);
// C++ 需要显式转换
char* cptr = static_cast<char*>(ptr);
```

### 3. **异常安全**
```cpp
// C 代码不知道 C++ 异常
extern "C" void c_function() {
    // 不要在这里抛出 C++ 异常
    // 异常不能跨越 C 函数边界
}
```

### 4. **资源管理**
```cpp
// 错误示例
void* c_data = c_lib_create();  // C 分配
delete c_data;                   // C++ 释放，危险！

// 正确做法
struct CDataWrapper {
    void* c_ptr;
    CDataWrapper() : c_ptr(c_lib_create()) {}
    ~CDataWrapper() { c_lib_free(c_ptr); }
    // 禁用拷贝
    CDataWrapper(const CDataWrapper&) = delete;
    CDataWrapper& operator=(const CDataWrapper&) = delete;
};
```

### 5. **标准库差异**
- C 的 `<string.h>` 和 C++ 的 `<string>` 完全不同
- 文件 I/O：C 的 `FILE*` 与 C++ 的 `fstream`

## 混合编程要达到的目标

### 1. **正确性目标**
- 无未定义行为
- 无内存泄漏
- 无悬垂指针
- 异常安全

### 2. **可维护性目标**
- 清晰的接口边界
- 良好的错误处理
- 可测试的模块
- 清晰的文档说明

### 3. **性能目标**
- 最小的调用开销
- 避免不必要的数据拷贝
- 缓存友好

### 4. **架构目标**
- 松耦合
- 高内聚
- 可独立测试

## 简单示例项目

### 项目 1：C 数学库的 C++ 包装
```cpp
// math_wrapper.h
#ifdef __cplusplus
extern "C" {
#endif

typedef struct Vector3 {
    float x, y, z;
} Vector3;

Vector3* vector_create(float x, float y, float z);
void vector_free(Vector3* v);
float vector_length(const Vector3* v);

#ifdef __cplusplus
}
#endif

// vector_wrapper.cpp
class Vector3Wrapper {
private:
    Vector3* m_vec;
    
public:
    Vector3Wrapper(float x, float y, float z) 
        : m_vec(vector_create(x, y, z)) {}
    
    ~Vector3Wrapper() { vector_free(m_vec); }
    
    float length() const { return vector_length(m_vec); }
    
    // 禁止拷贝
    Vector3Wrapper(const Vector3Wrapper&) = delete;
    Vector3Wrapper& operator=(const Vector3Wrapper&) = delete;
    
    // 允许移动
    Vector3Wrapper(Vector3Wrapper&& other) noexcept
        : m_vec(other.m_vec) { other.m_vec = nullptr; }
};
```

### 项目 2：回调函数集成
```cpp
// 在 C 中注册 C++ 回调
typedef void (*C_Callback)(void* user_data);

extern "C" void register_callback(C_Callback cb, void* data);

// C++ 侧
class CallbackHandler {
public:
    static void static_callback(void* data) {
        static_cast<CallbackHandler*>(data)->callback();
    }
    
    void callback() { /* C++ 逻辑 */ }
};
```

## 调试技巧
1. 使用 `nm` 或 `objdump` 查看符号
2. 在链接错误时检查名称修饰
3. 使用 `-g` 编译并配合 GDB/LLDB
4. 注意跨边界的栈展开

混合编程的关键是**清晰定义接口**和**严格管理资源**。从简单案例开始，逐步增加复杂度。
