C语言中的断言错误是指程序运行到`assert`宏时，其检查的条件为假（false），此时程序会立即终止并打印出错误信息。

**核心作用**：断言是一种调试工具，用于在开发阶段捕捉程序中“绝对不应该发生”的逻辑错误。

### 怎么用：assert宏
断言通过标准库宏 `assert` 实现，定义在 `<assert.h>` 头文件中。

**基本语法**： `assert(表达式);`
- 如果`表达式`的值为真（非零），则`assert`什么也不做，程序继续执行。
- 如果`表达式`的值为假（零），`assert`会：
    1.  向标准错误输出（通常是屏幕）打印一条错误信息，包含：失败的表达式、源文件名、行号。
    2.  调用 `abort()` 函数终止程序运行。

### 使用示例
```c
#include <stdio.h>
#include <assert.h>

int divide(int a, int b) {
    // 断言：除数b不能为0（这是一个程序逻辑上必须满足的前提条件）
    assert(b != 0);
    return a / b;
}

int main() {
    int result = divide(10, 2); // 正常执行，断言通过
    printf("Result: %d\n", result);

    result = divide(10, 0); // 程序运行至此会触发断言失败！
    // 将输出类似：Assertion failed: b != 0, file example.c, line 6
    // 随后程序终止。

    return 0;
}
```

### 关键注意事项
1.  **仅用于调试**：断言旨在发现开发者的编程错误，而非处理运行时可能发生的预期错误（如用户输入错误、文件打开失败等，这些应该用 `if` 判断和错误处理代码）。
2.  **发布版本中自动禁用**：在编译最终发布版本时，通常会在编译命令中定义 `NDEBUG` 宏。定义此宏后，所有的 `assert` 调用将被预处理为空的 `((void)0)`，不会产生任何代码和运行时开销。
    - 例如，使用GCC编译时：`gcc -DNDEBUG myprogram.c -o myprogram`
3.  **不要用断言检查必须处理的错误**：例如，不要用它来检查用户输入、malloc返回值或文件操作是否成功。这些是程序正常运行时可能出现的、必须妥善处理的状况。
4.  **断言表达式应无副作用**：确保断言内的表达式不会改变程序状态，因为在定义 `NDEBUG` 的发布版本中，该表达式不会被求值。

**总结**：使用`assert`来守护你确信为真的程序不变条件。它是一个强大的“急救员”，在开发阶段帮你快速定位假设不成立的内部逻辑错误。
