# C语言内联汇编核心笔记（GCC版）
## 一、内联汇编基础
### 1. 定义
在C代码中直接嵌入汇编指令，兼顾C的简洁性和汇编对硬件的精准控制（如底层驱动、性能优化、访问特殊寄存器），不同编译器语法不同（本文以GCC为主，区分x86和RISC-V架构）。

### 2. 核心语法框架（跨架构通用）
```c
asm volatile (
    "汇编指令1\n\t"  // \n\t：格式化换行缩进（非必需，提升可读性）
    "汇编指令2\n\t"
    : 输出操作数列表  // 汇编 → C：结果写回C变量
    : 输入操作数列表  // C → 汇编：读取C变量的值
    : 破坏列表       // 声明编译器未感知的寄存器/内存修改
);
```
- `volatile`：阻止编译器优化汇编代码（建议必加）；
- 架构差异：仅汇编指令/寄存器名不同（如x86用`eax`/`movl`，RISC-V用`a0`/`mv`），语法框架一致。

## 二、输入/输出操作数列表（C与汇编的桥梁）
### 1. 核心规则
| 类型       | 作用                          | 约束符前缀 | 示例                  |
|------------|-------------------------------|------------|-----------------------|
| 输出列表   | 汇编结果写回C变量             | `=`（只写）/`+`（读写） | `"=a"(res)`、`"+r"(num)` |
| 输入列表   | 汇编读取C变量的值             | 无         | `"r"(code)`、`"a"(eax)` |

### 2. 关键细节
- 占位符：`%0`/`%1`等按“输出在前、输入在后”编号，指代对应C变量；
- 常用约束符：
  - `r`：编译器自动选任意通用寄存器（推荐）；
  - `a`（x86）/`a0`（RISC-V）：指定特定寄存器；
  - `m`：直接操作变量内存地址（无寄存器中转）。

### 3. 示例
```c
// 输出列表：把eax的值写回res；输入列表：读取a/b的值做加法
int a=10, b=20, sum;
asm volatile (
    "movl %1, %%eax\n\t"
    "addl %2, %%eax\n\t"
    "movl %%eax, %0"
    : "=m"(sum)
    : "r"(a), "r"(b)
    : "%eax"
);
```

## 三、破坏列表（避免编译器优化冲突）
### 1. 定义
声明“汇编修改了但未出现在输入/输出列表中的寄存器/内存”，告诉编译器避开这些资源，防止数据冲突。

### 2. 必须加破坏列表的场景
1. 汇编修改了未声明的寄存器（如x86的`ebx`、RISC-V的`a1`）；
2. 汇编执行隐式修改寄存器的指令（如x86的`cpuid`）；
3. 汇编直接操作内存（需声明`"memory"`）。

### 3. 写法规则
- 寄存器：x86加`%`（如`"%ebx"`），RISC-V直接写（如`"a1"`）；
- 内存：固定写`"memory"`（强制寄存器与内存同步）；
- 多个项用逗号分隔：`"%eax", "%ebx", "memory"`。

### 4. "memory"约束的作用
- 执行汇编前：寄存器缓存的内存值写回内存；
- 执行汇编后：清空寄存器中过时的内存缓存（后续需重新读内存）；
- 禁止编译器重排序汇编前后的内存操作，避免数据不一致。

## 四、汇编指令分隔与格式化
### 1. `;` 与 `\n\t` 的区别
| 符号     | 作用                          | 必要性 |
|----------|-------------------------------|--------|
| `;`      | 汇编指令分隔符（汇编器层面）| 必需（单引号内多条指令） |
| `\n\t`   | 换行+缩进（格式化，提升可读性）| 非必需 |

### 2. 推荐写法（规范易读）
```c
// RISC-V示例：拆行拼接，用\n\t格式化，无;
asm volatile(
    "mv a0, %0\n\t"  // 第一条指令
    "li a1, 0x01\n\t"// 第二条指令
    "ebreak"         // 最后一条可省\n\t
    :
    : "r"(code)
    : "a1"
);
```

## 五、关键注意事项
1. 平台相关性：x86/ARM/RISC-V的汇编指令/寄存器完全不同，内联汇编会丧失可移植性；
2. 优化避坑：不加`volatile`可能导致编译器删除“无输出”的汇编；
3. 内存同步：直接操作内存时必须加`"memory"`约束，避免寄存器与内存数据不一致；
4. 调试难度：汇编指令无法直观调试，仅在必要时使用（如底层驱动、性能瓶颈优化）。
