# **Shell 脚本完全指南**

这份指南整合了核心语法、关键概念与最佳实践，旨在提供一份清晰、结构化的学习与速查手册。

## 1. 脚本的起点：Shebang
*   **定义**：脚本文件的第一行，格式为 `#!解释器路径`，用于告诉系统使用哪个解释器执行此脚本。
*   **关键规则**：
    *   必须是文件的**前两个字符**，不能有任何开头的空格。
    *   如果省略，系统会使用默认的Shell（通常`/bin/sh`）。
*   **实例**：
    ```bash
    #!/bin/bash           # 使用系统指定路径的Bash
    #!/usr/bin/python3    # 使用Python 3
    #!/usr/bin/env bash   # 通过env在PATH中查找bash，可移植性更好
    #!/usr/bin/env node   # 同上，用于Node.js
    ```

## 2. 执行脚本的方法
```bash
# 方法1：使用Shebang (最常见)
chmod +x myscript.sh    # 首先赋予执行权限
./myscript.sh           # 通过相对或绝对路径执行

# 方法2：手动指定解释器 (无需Shebang或执行权限)
bash myscript.sh
sh myscript.sh
python3 myscript.py
```

## 3. 基础语法核心

### 变量
*   **赋值**：`变量名=值` （**等号两侧绝不能有空格**）
    *   正例：`name="Alice"`
    *   反例：`name = "Alice"` （Shell会尝试运行名为`name`的命令）
*   **使用**：`$变量名`
*   **命令替换**：将命令输出捕获为变量值，推荐使用 `$()`。
    ```bash
    current_time=$(date)
    file_list=$(ls *.txt)  # 反引号 ` ` 也可用，但 $() 更现代，且支持嵌套
    ```

### 字符串引号
*   **单引号 `'`**：**原义字符串**。内部的所有字符（包括`$`, `` ` ``, `\`）都按字面意义解释。
    ```bash
    echo '$USER, today is `date`'  # 输出: $USER, today is `date`
    ```
*   **双引号 `"`**：**可插值字符串**。变量（`$VAR`）和命令替换（`$()`）会被展开。
    ```bash
    user="Bob"
    echo "Hello, $user"  # 输出: Hello, Bob
    ```

## 4. 核心控制流

### 条件判断 (if)
*   使用 `[[ ... ]]` 进行条件测试，它比 `[ ... ]` 或 `test` 更强大、更安全。
*   **基本结构**：
    ```bash
    if [[ 条件 ]]; then
        命令
    elif [[ 其他条件 ]]; then
        命令
    else
        命令
    fi
    ```
*   **常用测试运算符**：
    *   **字符串**：`-z "$str"` (为空), `-n "$str"` (非空), `"$s1" == "$s2"` (相等)
    *   **数字**：`$a -eq $b` (等于), `-ne` (不等), `-gt` (大于), `-lt` (小于)
    *   **文件**：`-f "file"` (是普通文件), `-d "dir"` (是目录), `-r "file"` (可读)
    *   **逻辑组合**：`&&` (与), `||` (或) 。在`[[ ]]`内使用更直观。
        ```bash
        if [[ -f "$file" && -r "$file" ]]; then
            echo "文件存在且可读"
        fi
        ```

### 循环
*   **`for` 循环** (遍历列表)：
    ```bash
    for fruit in apple banana cherry; do
        echo "I like $fruit"
    done
    # 遍历所有位置参数
    for arg in "$@"; do
        echo "Argument: $arg"
    done
    ```
*   **`while` 循环** (基于条件)：
    ```bash
    count=1
    while [[ $count -le 5 ]]; do
        echo "Count: $count"
        ((count++))  # 算术运算，使count自增1
    done
    ```

### 函数
*   **定义与调用**：
    ```bash
    # 定义
    greet() {
        local name=$1  # 使用local声明局部变量是个好习惯
        echo "Hello, $name!"
    }
    # 调用
    greet "World"  # 输出: Hello, World!
    ```
*   **参数**：函数内部通过 `$1`, `$2`, `$3`... 访问传入的参数。

## 5. 特殊变量与退出码
| 变量 | 含义 |
| :--- | :--- |
| `$0` | 当前脚本或Shell的名称。 |
| `$1` - `$9` | 第1到第9个位置参数。 |
| `$#` | 传递给脚本或函数的位置参数**个数**。 |
| `$@` | 所有位置参数的列表（每个参数独立，推荐用于遍历）。 |
| `$*` | 所有位置参数合并为单个字符串（与`$@`有细微差别）。 |
| `$?` | **上一个命令的退出状态码**。`0`=成功，非`0`=失败。 |
| `$$` | 当前Shell的进程ID (PID)。 |
| `!!` | 上一条完整的命令。快速重试：`sudo !!` |
| `$_` | 上一条命令的最后一个参数。 |

## 6. 逻辑与命令组合
基于退出码 (`$?`) 控制流程：

*   **`CMD1 && CMD2`** (逻辑与)：`CMD1`**成功**（返回`0`）**后才执行**`CMD2`。
    ```bash
    mkdir new_dir && cd new_dir  # 只有目录创建成功，才会进入
    ```
*   **`CMD1 || CMD2`** (逻辑或)：`CMD1`**失败**（返回非`0`）**后才执行**`CMD2`。
    ```bash
    cp important.txt backup/ || echo "备份失败！"
    ```
*   **`CMD1 ; CMD2`** (顺序执行)：无论`CMD1`成功与否，都执行`CMD2`。

## 7. 通配与花括号展开
*   **通配符 (Globbing)**：
    *   `?`：匹配任意单个字符。 (`file?.txt` 匹配 `file1.txt`, `fileA.txt`)
    *   `*`：匹配任意数量（包括零个）的任意字符。 (`*.log` 匹配所有 `.log` 文件)
*   **花括号展开 `{}`**：生成任意组合，**不检查文件是否存在**。
    ```bash
    echo {A,B,C}-{1,2}    # 输出: A-1 A-2 B-1 B-2 C-1 C-2
    mkdir -p {2020..2023}-{01..12}  # 创建2020-01到2023-12的目录
    cp script.sh{,.bak}   # 大括号展开为 `cp script.sh script.sh.bak`
    ```

## 8. 高级特性

### 进程替换 `<()`
将命令的输出“伪装”成一个临时文件（命名管道）的路径，供其他需要文件参数的命令使用。
```bash
# 比较两个目录下的文件列表差异
diff <(ls dir1) <(ls dir2)
# 将 grep 的结果作为一个“文件”提供给另一个命令
grep -l "error" <(find . -name "*.log")
```

## 9. 脚本 vs. 函数
| 方面 | Shell 函数 | Shell 脚本 |
| :--- | :--- | :--- |
| **定义** | 在Shell进程内定义 (`func_name() { ... }`) | 独立的文本文件 |
| **语言** | 必须与当前Shell语言一致 | 任意语言（由Shebang指定） |
| **执行环境** | 在**当前Shell进程**中运行，可修改环境（如`cd`、变量） | 在**新的子进程**中运行，默认不影响父Shell环境（除非用`source`或`.`执行） |
| **加载速度** | 定义时即加载，很快 | 每次执行需读取文件，相对较慢 |
| **用途** | 封装常用操作，简化命令行 | 复杂的、可重复执行的任务 |

## 10. 最佳实践与调试
*   **错误处理**：在脚本开头使用 `set -euo pipefail`。
    *   `-e`：任何命令失败（非零退出码）则立即退出脚本。
    *   `-u`：遇到未定义的变量时报错并退出。
    *   `-o pipefail`：管道中任意命令失败，整个管道都视为失败。
*   **调试输出**：在脚本中任何地方使用 `set -x` 开启命令追踪，`set +x` 关闭。
*   **代码检查**：使用在线工具 **https://www.shellcheck.net/** 检查脚本语法和常见陷阱。
*   **重定向**：
    ```bash
    command > output.log 2>&1  # 标准输出和错误都重定向到同一文件
    command &> /dev/null       # 丢弃所有输出（Bash简写形式）
    command 2> error.log       # 仅将标准错误重定向到文件
    ```

## 核心思想总结
Shell脚本的核心优势在于**无缝衔接和操控命令行工具与系统资源**。掌握变量、引号、控制流和特殊变量是基础，理解命令的退出码和逻辑操作符是编写健壮脚本的关键。通过函数和脚本来封装复杂操作，并利用通配、展开和进程替换等特性可以极大地提升效率。
