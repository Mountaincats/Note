### 一、文件检索工具
1.  **`cat`** (concatenate)
    *   **作用**：连接文件并打印到标准输出。主要用于查看小文件内容、合并多个文件。
    *   **常用选项**：
        *   `-n`：显示行号。
        *   `-A`：显示所有字符，包括制表符（`^I`）和行尾符号（`$`）。
    *   **示例**：
        ```bash
        cat file.txt                 # 查看文件
        cat -n script.sh             # 带行号查看脚本
        cat file1.txt file2.txt > combined.txt  # 合并文件
        ```

2.  **`more` / `less`**
    *   **作用**：分页查看大文件内容，避免内容快速滚过屏幕。`less` 比 `more` 功能更强，支持向前/后滚动、搜索等。
    *   **常用选项** (`less`)：
        *   交互命令：`空格键`（向下翻页），`f``b``d``u`（上下翻页），`g`（跳到开头），`G`（跳到结尾），`/关键词`（搜索），`q`（退出）。
        *   启动选项：`-N` 显示行号，`+F` 实时跟踪文件尾部（类似 `tail -f`）。
    *   **示例**：
        ```bash
        less -N large_log.log        # 带行号查看日志
        more +20 file.txt            # 从第20行开始显示
        less +F /var/log/syslog      # 实时追踪日志
        ```

3.  **`head` / `tail`**
    *   **作用**：显示文件的开头（`head`）或结尾（`tail`）部分内容。默认显示10行。
    *   **常用选项**：
        *   `-n <数字>`：指定显示的行数，如 `-n 5`。
        *   `-f` (`tail`特有)：持续输出文件新增的内容，常用于监控日志。按 `Ctrl+C` 退出。
        *   `-c <数字>`：指定显示的字节数。
    *   **示例**：
        ```bash
        head -n 3 config.json        # 查看配置文件前3行
        tail -f -n 20 app.log        # 实时监控日志最后20行及新增内容
        tail -c 100 data.bin         # 查看文件最后100个字节
        ```

4.  **`file`**
    *   **作用**：检测文件的类型。通过分析文件内容（而非仅后缀名）来判断是文本文件、二进制可执行文件、压缩包、图片等。
    *   **常用选项**：`-i` 输出MIME类型字符串。
    *   **示例**：
        ```bash
        file picture.png            # 输出：PNG image data, 1920 x 1080, ...
        file -i document.pdf         # 输出：application/pdf; charset=binary
        file unknown.bin             # 常用于判断未知文件格式
        ```

5.  **`find`**
    *   **作用**：在指定目录树中递归地查找文件，并可根据文件名、类型、大小、修改时间、权限等丰富条件进行筛选，对结果执行操作。
    *   **常用选项**：
        *   `-name`：按文件名匹配（支持`Shell 通配符` `*`, `?`, `[]`）。
        *   `-iname`：不区分大小写，其余同`-name`。
        *   `-regex`：匹配的是从 `find` 起始路径开始的完整相对路径，而不仅仅是文件名（支持正则表达式）
        *   `-iregex`：不区分大小写，其余同`-regex`
        *   `-type`：按文件类型查找。`f`（普通文件），`d`（目录），`l`（符号链接）。
        *   `-size`：按文件大小查找。如 `+10M`（大于10MB），`-1G`（小于1GB）。
        *   `-mtime` / `-atime`：按修改/访问时间查找。`-7`（7天内），`+30`（30天前）。
        *   `-exec`：对查找到的每个文件执行命令。`{}` 代表当前文件名，`\;` 结束命令。
        *   `-delete`：删除匹配的文件。
        *   `-maxdepth <num>`：查找深度
    *   **示例**：
        ```bash
        find . -name "*.tmp" -delete                    # 删除当前目录下所有.tmp文件
        find /home -type f -size +100M -exec ls -lh {} \; # 查找大于100MB的文件并列出详情
        find /var/log -name "*.log" -mtime -1           # 查找/var/log下一天内修改过的日志文件
        ```

### 二、输入输出控制
1. **重定向**
    *   **作用**：控制命令的输入来源和输出目标。
    *   **语法**：**`命令 [n] 操作符 目标`**
        *   **命令**： 要执行的命令或程序。
        *   **文件描述符 `[n]` (可选)**： 一个数字，代表数据流。**省略时，默认是 `1`（标准输出）。**
            *   `0` - 标准输入 (`stdin`)
            *   `1` - 标准输出 (`stdout`)
            *   `2` - 标准错误 (`stderr`)
        *   **操作符**： 决定重定向的行为（覆盖、追加、输入等）。
        *   **目标**： 通常是一个文件，也可以是特殊文件（如 `/dev/null`）或其他文件描述符。
    *   **详细语法说明**：
        *   **`>` 或 `1>`**：将**标准输出**覆盖写入到文件，例如 `ls > list.txt`
        *   **`>>` 或 `1>>`**：将**标准输出**追加到文件末尾，例如 `echo "New line" >> log.txt`
        *   **`2>`**：将**标准错误**覆盖写入到文件，例如 `rm nofile 2> error.log`
        *   **`2>>`**：将**标准错误**追加到文件末尾，例如 `make 2>> errors.log`
        *   **`&>`**：将**标准输出和标准错误**都覆盖重定向到同一处 (Bash 简写)，例如 `command &> all_output.log`
        *   **`&>>`**：将**标准输出和标准错误**都追加到同一处，例如 `command &>> all_output.log`
        *   **`2>&1`**：将**标准错误**合并到**标准输出**所在的同一目标，例如 `command > output.log 2>&1`
        *   **`<`**：从文件读取内容作为命令的**标准输入**，例如 `sort < input.txt`
        *   **`<<`**：Here Document，用于内联输入，例如 `cat << EOF`
    *   **关键点**：
        *   **顺序很重要**：在 `command > file 2>&1` 中，先建立标准输出到文件的重定向，然后将标准错误指向同一目标。写成 `command 2>&1 > file` 则效果不同。
        *   **`&>` 与 `2>&1`**：在简单重定向到文件时，`&>` 是更简洁的写法。在管道操作等复杂场景中，`2>&1` 的写法更通用。

2.  **管道 `|`**
    *   **作用**：将一个命令的标准输出作为下一个命令的标准输入。是组合简单命令完成复杂任务的核心。
    *   **示例**：
        ```bash
        ps aux | grep nginx          # 查找所有包含“nginx”的进程
        cat /var/log/*.log | grep "ERROR" | wc -l  # 统计所有日志中的ERROR行数
        ```

3.  **`tee`**
    *   **作用**：从标准输入读取数据，**同时**输出到标准输出和一个或多个文件中。常用于在查看管道数据的同时进行保存。
    *   **常用选项**：`-a` 以追加模式写入文件，而非覆盖。
    *   **示例**：
        ```bash
        ./configure | tee config.log  # 显示配置过程并保存到日志
        echo "备份开始" | tee -a backup.log  # 在屏幕显示并追加到日志
        ```

4.  **`xargs`**
    *   **作用**：将管道或标准输入传递来的数据（通常是一长串文件名或其他参数）转换成命令行参数，传递给后续命令。解决`find -exec`或命令参数过长的问题。
    *   **常用选项**：
        *   `-n <数字>`：每次命令使用几个参数。`echo {1..10} | xargs -n 3 echo`
        *   `-I {}`：使用占位符替换，通常用于需要将参数插入命令中间的情况。`find . -name "*.bak" | xargs -I {} rm {}`
        *   `-P <数字>`：并行运行多个进程，加速处理。`find . -name "*.jpg" | xargs -P 4 -I {} convert {} {}.png`
        *   `-0` 或 `--null`：与 `find -print0` 配合，处理包含空格、换行的文件名。
    *   **示例**：
        ```bash
        find . -name "*.log" -print0 | xargs -0 rm        # 安全删除所有日志文件
        cat urls.txt | xargs -P 8 -n 1 wget -q            # 并行下载多个URL
        ```

### 三、文本处理工具
1.  **`grep`** (global regular expression print)
    *   **作用**：在文本中搜索匹配指定模式的行。是文本过滤和搜索的瑞士军刀。
    *   **常用选项**：
        *   `-i`：忽略大小写。
        *   `-I`：跳过二进制文件，只找文本文件。
        *   `-v`：反向选择，即显示**不包含**模式的行。
        *   `-n`：显示匹配行的行号。
        *   `-c`：只显示匹配行的总数。
        *   `-r` 或 `-R`：递归搜索目录及其子目录。
        *   `-l`：只显示包含匹配项的文件名，而非具体行。
        *   `-E`：使用扩展正则表达式（ERE）。**（重要）**
        *   `-P`：使用Perl兼容的正则表达式（PCRE），功能最强大。**（如果支持）**
    *   **正则支持说明**：
        *   默认使用**基本正则表达式 (BRE)**。
        *   使用 `-E` 选项启用**扩展正则表达式 (ERE)**，支持 `+`, `?`, `|`, `()` 等元字符。
        *   使用 `-P` 选项启用**Perl正则 (PCRE)**，支持 `\d`, `\w`, 向前/向后断言等高级特性。
    *   **示例**：
        ```bash
        grep -n "error" app.log                     # 显示带行号的错误行
        grep -E "^[A-Z]" file.txt                  # 使用ERE，匹配以大写字母开头的行
        grep -P "\d{3}-\d{4}" contacts.txt         # 使用PCRE，匹配电话号码
        grep -r --include="*.c" "TODO" src/         # 递归搜索src目录下所有.c文件的TODO
        ```
    *   **相似工具**：`ripgrep` (`rg`)，速度更快，默认递归，彩色输出。

2.  **`awk`** (Aho, Weinberger, Kernighan)
    *   **作用**：强大的文本分析处理语言。按行处理，并自动将每行按分隔符（默认空格）分割成若干字段（`$1`, `$2`, ... `$NF` 表示最后一个字段，`$0` 表示整行）。支持变量、数组、控制流、算术和字符串运算。
    *   **常用选项/变量**：
        *   `-F`：指定输入字段分隔符。`-F:` 表示用冒号分隔。
        *   `-v var=value`：定义变量。
        *   `BEGIN{...}`：在处理任何输入行之前执行。
        *   `END{...}`：在处理完所有输入行之后执行。
        *   `NR`：当前处理的总行号（记录号）。
        *   `NF`：当前行的字段数量。
    *   **正则支持说明**：`awk` 本身支持扩展正则表达式（ERE）。模式匹配语法为 `~`（匹配）和 `!~`（不匹配）。
    *   **示例**：
        ```bash
        awk '{print NR, $1, $NF}' data.txt              # 打印每行的行号、第一列和最后一列
        awk -F: '$3 >= 1000 {print $1}' /etc/passwd      # 打印UID大于等于1000的用户名
        awk '/^Error/ {count++} END {print count}' log  # 统计以Error开头的行数
        ls -l | awk '{sum+=$5} END {print "Total:", sum}' # 计算当前目录下文件总大小
        ```

3. **`sed`** (stream editor)
   *   **作用**：流编辑器，用于对输入流（文件或管道）进行基本的文本转换（替换、删除、插入、选取等）。尤其擅长**按行**处理。
   *   **常用选项**：
       *   `-n`：静默模式，仅显示经过特殊处理（如`p`命令）的行。
       *   `-e`：直接在命令行上进行编辑操作，可接多个`-e`实现多步编辑。
       *   `-i`：**直接修改文件内容**（危险！建议先测试或用`-i.bak`创建备份）。
       *   `-r` 或 `-E`：使用扩展正则表达式（ERE），GNU sed优先用`-r`，BSD/macOS sed优先用`-E`。
       *   `-f`：从指定文件中读取编辑脚本（批量执行sed命令）。
       *   `-u`：无缓冲输出，实时处理并打印结果（适合大文件/流处理）。
   *   **常用命令**：
       *   `s/pattern/replacement/flags`：替换命令，最常用。
           - 标志：`g`（全局替换）、`n`（替换第n次匹配）、`p`（打印替换后的行）、`i`（忽略大小写，GNU sed）、`w file`（将替换结果写入文件）。
       *   `d`：删除匹配的行（如`sed '/^#/d' file`删除注释行）。
       *   `p`：打印匹配的行，常与`-n`联用（避免重复输出）。
       *   `a\text`：在匹配行**后**追加文本（如`sed '/EOF/a\exit' file`）。
       *   `i\text`：在匹配行**前**插入文本（如`sed '/start/i\init' file`）。
       *   `c\text`：替换整行内容（如`sed '/error/c\ERROR: failed' file`）。
       *   `y/source/dest/`：字符转换（一对一替换，如`sed 'y/abc/123/' file`将a→1、b→2、c→3）。
       *   `=`：打印行号（如`sed -n '/error/=' file`显示错误行的行号）。
       *   `q`：匹配后退出（如`sed '/first/q' file`找到first行后停止处理）。
   *   **正则支持说明**：
       *   默认使用**基本正则表达式 (BRE)**：特殊字符（`+`、`?`、`|`、`()`）需转义为`\+`、`\?`、`\|`、`\(\)`。
       *   使用 `-r` (GNU sed) 或 `-E` (BSD sed) 启用**扩展正则表达式 (ERE)**：无需转义特殊字符，语法更简洁。
       *   通配符：`^`（行首）、`$`（行尾）、`.`（任意单个字符）、`*`（前字符0+次）、`[]`（字符集）、`[^]`（非字符集）。
   *   **示例**：
       ```bash
       # 基础替换
       sed 's/foo/bar/g' file.txt                # 将所有foo替换为bar（仅输出到屏幕）
       sed -i.bak 's/old/new/g' file.txt         # 直接修改文件，并创建备份file.txt.bak
       sed 's/foo/bar/2' file.txt                # 仅替换每行中第2个foo为bar
       sed 's/^/    /' file.txt                  # 给所有行添加4个空格（缩进）
       sed 's,$, # comment,' file.txt            # 用,做分隔符，给每行末尾加注释（避免转义/）
       
       # 匹配与打印
       sed -n '/^Error/p' log.txt                # 打印所有以Error开头的行
       sed -n '/warning/{p;q}' log.txt           # 打印第一个warning行后退出
       sed -n '10,20p' file.txt                  # 打印第10-20行
       
       # 正则与分组
       sed -r 's/([0-9]+)/<\1>/g' data.txt       # 用<>将数字括起来（ERE分组，\1引用第一个分组）
       sed 's,\(maze\)\.o[ :]*,\1.o maze.d : ,g' < maze.d.$$ > maze.d  # 替换编译依赖，用,分隔避免转义/
       
       # 删除与插入
       sed '/^$/d' file.txt                      # 删除空行
       sed '3d' file.txt                         # 删除第3行
       sed '/temp/a\=====' file.txt              # 在包含temp的行后插入分隔符
       sed '1i\# Auto-generated file' file.txt   # 在第一行前插入注释
       
       # 多命令组合
       sed -e 's/foo/bar/g' -e '/test/d' file.txt # 先全局替换foo→bar，再删除含test的行
       sed -f script.sed file.txt                # 执行脚本文件中的sed命令（script.sed每行一个sed命令）
       ```
   *   **注意**：
       1. 在 sed的 `s` 命令中，分隔符可以是任意字符（`/`、`#`、`,`、`@`等），当模式或替换内容中包含`/`时，优先用其他分隔符避免转义（如`sed 's#/usr/bin#/bin#' file`）。
       2. `-i` 选项在不同系统行为略有差异：GNU sed支持`-i''`（无备份直接修改），BSD/macOS sed需写`-i ''`（空格不可省略）。
       3. 处理特殊字符（如`&`、`\`）：`&`代表匹配的整段内容（`sed 's/error/[&]/g'`→`[error]`），`\`需转义为`\\`。
       4. 管道处理：sed可无缝对接其他命令（如`cat file.txt | grep error | sed 's/error/ERROR/g'`）。

### 总结
1. `sed` 核心是**按行处理文本**，最常用场景是替换（`s`命令）、筛选（`-n`+`p`）、删除（`d`），`-i`直接修改文件需谨慎（建议先备份）。
2. 正则表达式默认用BRE，加`-r/-E`切换为ERE可简化语法，`s`命令的分隔符可灵活替换以避免转义`/`。
3. 多步编辑可通过`-e`（命令行）或`-f`（脚本文件）实现，结合管道可完成复杂的文本处理流水线。

4.  **`sort`** / **`uniq`** / **`wc`** (word count)
    *   **作用**：
        *   `sort`：对文本行进行排序。
        *   `uniq`：报告或忽略连续重复的行。**注意：`uniq` 通常需要输入是排序好的，或与 `sort` 结合使用。**
        *   `wc`：统计文件的行数、单词数和字节数。
    *   **常用选项 (`sort`)**：
        *   `-n`：按数值大小排序。
        *   `-r`：反向排序。
        *   `-k`：指定排序的列（字段）。`-k 2` 按第二列，`-k 3n` 按第三列数值排序。
        *   `-t`：指定字段分隔符。
    *   **常用选项 (`uniq`)**：
        *   `-c`：在每行前显示该行重复出现的次数。
        *   `-d`：仅显示重复出现的行。
        *   `-u`：仅显示不重复的行。
    *   **常用选项 (`wc`)**：
        *   `-l`：只统计行数。
        *   `-w`：只统计单词数。
        *   `-c`：只统计字节数。
        *   `-m`：只统计字符数（与字节数在多字节编码下不同）。
    *   **示例**：
        ```bash
        sort -t, -k2 -n data.csv                # 按逗号分隔的第二列数值排序
        sort file.txt | uniq -c | sort -nr      # 统计每行出现次数并按次数降序排列
        wc -l access.log                        # 统计日志文件行数
        ```

5.  **`cut`** / **`tr`** (translate)
    *   **作用**：
        *   `cut`：从文件的每一行中剪切出部分（字段、字符、字节）。
        *   `tr`：替换或删除字符。**注意：`tr` 处理的是字符，不是更复杂的模式。**
    *   **常用选项 (`cut`)**：
        *   `-d`：指定字段分隔符（默认为TAB）。
        *   `-f`：选择指定字段。`-f 1,3` 选择第1和3列，`-f 2-5` 选择2到5列。
        *   `-c`：选择指定字符。`-c 1-10` 选择每行的1-10个字符。
    *   **常用选项 (`tr`)**：
        *   `SET1` `SET2`：将 `SET1` 中的字符替换为 `SET2` 中对应位置的字符。
        *   `-d`：删除 `SET1` 中出现的所有字符。
        *   `-s`：将 `SET1` 中重复出现的字符压缩为一个。
    *   **示例**：
        ```bash
        cut -d: -f1,7 /etc/passwd                # 提取用户名和登录shell
        echo "HELLO" | tr '[:upper:]' '[:lower:]' # 转换为小写
        echo "hello  world" | tr -s ' '          # 将连续空格压缩为一个
        cat file.txt | tr -d '\r' > newfile.txt  # 删除Windows换行符^M (\r)
        ```

6.  **`indent`**
    *   **作用**：C/C++ 源代码格式化工具，自动调整缩进、括号位置、空格等，使代码风格统一。
    *   **常用选项**（GNU风格为例）：
        *   `-kr` 或 `-k&r`：使用 Kernighan & Ritchie (K&R) 风格。
        *   `-i4`：缩进使用4个空格。
        *   `-nut` 或 `--no-tabs`：使用空格，而非制表符。
        *   `-l80` 或 `--line-length80`：设置最大行长为80个字符。
        *   `-orig` 或 `--original`：使用原始伯克利风格。
    *   **示例**：
        ```bash
        indent -kr -i4 -l80 -nut messy_code.c -o formatted_code.c
        ```

### 四、正则表达式（Regex）支持回顾
*   **POSIX 基本正则表达式 (BRE)**：`grep` (默认), `sed` (默认)。`+`, `?`, `|`, `()`, `{}` 需转义才具特殊含义。
*   **POSIX 扩展正则表达式 (ERE)**：`grep -E`, `sed -r/-E`, `awk`。上述元字符无需转义。
*   **Perl兼容正则表达式 (PCRE)**：`grep -P` (GNU grep)，功能最强大，支持 `\d`, `\w`, 向前/后断言等。非所有工具支持。

### 五、系统监控与进程管理工具
1.  **`jobs`** / **`bg`** / **`fg`**
    *   **作用**：管理**作业**（jobs），即shell中运行的任务（进程组），常用于前后台切换。
    *   **使用流程**：
        1.  运行一个命令，如 `sleep 100`。
        2.  按 `Ctrl+Z`：将**前台**任务暂停，并放入后台。
        3.  输入 `jobs`：查看当前所有后台作业及其状态和编号。
        4.  输入 `bg %1`：将1号作业转为**后台继续运行**。
        5.  输入 `fg %1`：将1号作业调回**前台运行**。

2.  **`ps`** (process status)
    *   **作用**：显示当前进程状态的一个**快照**。功能极其丰富，选项风格多样（BSD, UNIX, GNU）。
    *   **常用选项组合**：
        *   `ps aux`：查看系统所有进程的详细信息（BSD风格）。`a`(所有终端)，`u`(用户格式)，`x`(无控制终端进程)。
        *   `ps -ef`：以完整格式列表显示所有进程（UNIX风格）。
        *   `ps -eLf`：显示进程及其下的线程（`LWP`列）。
        *   `ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%mem`：自定义输出列并按内存排序。
    *   **示例**：
        ```bash
        ps aux | grep python             # 查找所有Python进程
        ps -p 1234 -o pid,ppid,cmd       # 查看指定PID进程的父进程
        ```

3.  **`top`** / **`htop`**
    *   **作用**：动态、实时地显示系统中进程的资源占用状况（CPU、内存等）。`htop` 是 `top` 的增强版，界面更友好，支持鼠标、颜色、树状视图。
    *   **常用交互命令 (`top`)**：
        *   `P`：按CPU使用率排序。
        *   `M`：按内存使用率排序。
        *   `T`：按运行时间排序。
        *   `k`：终止一个进程（输入PID）。
        *   `1`：显示所有CPU核心的详情。
        *   `q`：退出。
    *   **示例**：直接运行 `top` 或 `htop` 即可。

4.  **`kill`** / **`pkill`**
    *   **作用**：向进程发送信号。默认发送 `SIGTERM` (15)，请求进程终止。`-9` (`SIGKILL`) 强制终止。
    *   **常用信号**：
        *   `-1` 或 `-HUP`：挂起，通常用于让守护进程重新读取配置文件。
        *   `-2` 或 `-INT`：中断 (Ctrl+C)。
        *   `-9` 或 `-KILL`：强制杀死，进程无法捕获或忽略。
        *   `-15` 或 `-TERM`：正常终止（默认）。
    *   **示例**：
        ```bash
        kill 1234                    # 发送TERM信号给PID 1234
        kill -9 5678                 # 强制杀死PID 5678
        pkill -f "python script.py"  # 杀死所有命令行匹配"python script.py"的进程
        pkill -u username firefox    # 杀死用户username的firefox进程
        ```

5.  **`free`**
    *   **作用**：显示系统内存（物理内存和交换空间）的使用情况。
    *   **常用选项**：`-h` 以人类可读的单位（GiB, MiB）显示。`-m` 以MB为单位。
    *   **关键指标**：
        *   `total`：总量。
        *   `used`：已使用（包含cached/buffers）。
        *   `free`：完全未使用的。
        *   `available`：**估计**可用于启动新应用的内存量（包含可回收的缓存）。**这是关键参考值**。
    *   **示例**：
        ```bash
        free -h
        ```

6.  **`dmesg`**
    *   **作用**：显示和控制内核环形缓冲区（ring buffer）中的消息。这些消息包含了内核启动、硬件检测、设备驱动加载、错误等信息。是排查硬件和驱动问题的首要工具。
    *   **常用选项**：`-T` 显示人类可读的时间戳，`-H` 在 `less` 查看器中高亮显示。
    *   **示例**：
        ```bash
        dmesg | grep -i "error\|fail"      # 查看内核错误
        dmesg | grep -i "usb\|eth0"         # 查看USB或网卡相关日志
        dmesg -T | tail -20                # 查看最近20条内核日志，带时间
        ```

7.  **rsyslog与journald**
    *   **作用**：当系统启动到一定阶段，初始化系统会启动 `rsyslogd` 或 `systemd-journald` 服务。这些日志服务启动后，会立刻去读取环形缓冲区里已有的内容，将其存入自己的存储（文件或数据库），并开始持续监听缓冲区，将后续的新内核消息也实时转存下来

8.  **`lsof`** (list open files)
    *   **作用**：列出系统中**被打开的文件**以及打开它们的**进程**。在Linux中，一切皆文件（包括网络连接、设备、管道等），因此此命令用途极广。
    *   **常用选项**：
        *   `lsof [文件名]`：查看谁打开了这个文件。
        *   `-i`：列出所有网络连接。`-i:80` 列出使用80端口的连接。
        *   `-p [PID]`：列出指定进程打开的所有文件。
        *   `-u [用户名]`：列出指定用户打开的所有文件。
        *   `+D [目录]`：递归列出指定目录下所有被打开的文件。
    *   **示例**：
        ```bash
        lsof /var/log/syslog        # 查看哪个进程正在读写系统日志
        lsof -i :3306               # 查看哪个进程在使用MySQL默认端口
        lsof -p 1234                # 查看PID为1234的进程打开了哪些文件
        lsof -c sshd                # 查看所有sshd进程打开的文件
        ```

### 六、其它实用工具
1.  **`objdump`**
    *   **作用**：显示目标文件（如可执行文件、目标文件、库）的详细信息，包括反汇编代码、节头、符号表等。是逆向工程和调试的利器。
    *   **常用选项**：
        *   `-d`：**反汇编**可执行节（即代码段）。
        *   `-D`：反汇编所有节。
        *   `-S`：在反汇编代码中**交叉显示源代码**（需要编译时带`-g`调试信息）。
        *   `-h`：显示节头（section headers）信息。
        *   `-t`：显示符号表。
    *   **示例**：
        ```bash
        objdump -d ./myprogram > disassembly.txt
        objdump -S ./myprogram > source_and_asm.txt
        objdump -t ./mylib.so | grep "main"  # 在动态库中查找main符号
        ```

2.  **`time`**
    *   **作用**：测量命令执行所消耗的时间（真实时间、用户态CPU时间、内核态CPU时间）。注意：shell有内置的`time`命令，功能简单；外部命令`/usr/bin/time`（通常写作`/usr/bin/time`或`command time`）提供更详细的资源统计。
    *   **常用用法**（外部命令）：
        *   `/usr/bin/time -v command`：显示详细的资源使用统计，如最大内存占用、页面错误、I/O等。
    *   **示例**：
        ```bash
        time ls -R / > /dev/null           # 简单计时
        /usr/bin/time -v ./my_heavy_program  # 详细资源使用报告
        ```

3.  **`du`** (disk usage)
    *   **作用**：估算文件或目录占用的磁盘空间。
    *   **常用选项**：
        *   `-h`：以人类可读单位显示（K, M, G）。
        *   `-s`：只显示总计，不显示子目录详情。
        *   `-c`：最后给出所有参数的总计。
        *   `--max-depth=N`：深入到第N层目录。`--max-depth=1` 显示当前目录下第一级子目录大小。
        *   `-a`：显示所有文件的大小，而不仅仅是目录。
    *   **示例**：
        ```bash
        du -sh /home/*                     # 查看/home下各用户目录大小
        du -sh .                           # 查看当前目录总大小
        du -h --max-depth=1 /usr/share     # 查看/usr/share下一级子目录大小
        find . -name "*.log" -exec du -ch {} + | tail -1  # 计算所有.log文件总大小
        ```

4.  **`history`**
    *   **作用**：显示或操作当前shell会话的命令历史记录。历史记录通常保存在 `~/.bash_history` (Bash) 或 `~/.zsh_history` (Zsh) 中。
    *   **常用选项/变量**：
        *   `history N`：显示最近的N条命令。
        *   `history | grep keyword`：搜索包含关键字的命令。
        *   `!N`：执行历史记录中第N条命令。
        *   `!!`：执行上一条命令。
        *   `!keyword`：执行最近一条以keyword开头的命令。
        *   `history -c`：**清空当前shell会话的历史记录**（不影响历史文件）。
    *   **示例**：
        ```bash
        history 20                         # 查看最近20条命令
        history | grep ssh                # 查找所有包含ssh的命令
        !!                                # 重新执行上一条命令
        !vim                              # 执行最近一条以vim开头的命令
        ```

5.  **`chmod`** (change mode)
    *   **作用**：修改文件或目录的访问权限。
    *   **权限表示**：
        *   **字母表示法**：`u`(用户)，`g`(组)，`o`(其他)，`a`(全部)；`+`(添加)，`-`(移除)，`=`(设定)；`r`(读, 4)，`w`(写, 2)，`x`(执行/进入, 1)。
        *   **数字表示法**：三位八进制数，分别代表用户、组、其他用户的权限之和。如`755` = `rwxr-xr-x`。
    *   **示例**：
        ```bash
        chmod u+x script.sh                # 给文件所有者添加执行权限
        chmod go-w important.txt          # 移除组和其他用户的写权限
        chmod 600 ~/.ssh/id_rsa           # 私钥文件，仅所有者可读写
        chmod 755 /usr/local/bin/mytool    # 所有者可读写执行，其他人可读执行
        chmod -R a+rX public_dir/         # 递归设置，X表示只对目录设置执行权限
        ```

6.  **`ln`** (link)
    *   **作用**：创建文件的链接。链接分为**硬链接**（hard link）和**软链接**（symbolic link，symlink）。
    *   **区别**：
        *   **硬链接** (`ln source target`)：与源文件共用inode和数据块，是同一文件的另一个名字。删除源文件不影响硬链接。不能跨文件系统，不能链接目录。
        *   **软链接/符号链接** (`ln -s source target`)：创建一个特殊的文件，其内容是指向源文件的路径。类似于Windows的快捷方式。可以跨文件系统，可以链接目录。源文件被删除后，软链接会“断链”。
    *   **示例**：
        ```bash
        ln important.txt backup.txt        # 创建硬链接
        ln -s /mnt/shared ~/Desktop/shared # 创建指向共享目录的软链接
        ls -l ~/Desktop/shared            # 查看时会显示 `shared -> /mnt/shared`
        ```

7.  **`ping`**
    *   **作用**：向网络主机发送ICMP ECHO_REQUEST数据包，测试网络连通性、延迟和丢包率。
    *   **常用选项**：
        *   `-c N`：发送N个数据包后停止。`ping -c 4 www.baidu.com`
        *   `-i N`：设置发送间隔为N秒。`ping -i 0.5`
        *   `-s N`：设置发送数据包的大小为N字节。
        *   `-t N` (Linux下是`-w`或`-W`，具体查看man)：设置超时时间。
    *   **示例**：
        ```bash
        ping -c 5 -i 0.2 8.8.8.8         # 向谷歌DNS发送5个包，间隔0.2秒
        ```

    ```bash
    find . -name "*.py" -o -name "*.java" | xargs wc -l
    ```

    ```bash
    lsof -i :80
    # 或
    netstat -tlnp | grep :80
    ```

### 八、其它技巧
1.  **隐藏历史命令**：
    在Bash中，设置 `HISTCONTROL=ignorespace` 或 `HISTCONTROL=ignoreboth` 到 `~/.bashrc` 文件。在Zsh中，设置 `setopt HIST_IGNORE_SPACE` 到 `~/.zshrc` 文件。之后，在命令开头加上一个**空格**，该命令就不会被记录到历史中。
    ```bash
    # 在~/.bashrc中添加
    export HISTCONTROL=ignorespace
    # 然后 source ~/.bashrc
    # 之后，带空格的命令不会被记录
     mysupersecretcommand
    ```
2.  **使用 `!!` 和 `!$` 快速重复命令**：
    *   `!!`：上一条命令。
    *   `!$`：上一条命令的**最后一个参数**。
    ```bash
    mkdir -p /a/very/long/path/to/newdir
    cd !$  # 执行后变为 cd /a/very/long/path/to/newdir
    ```
