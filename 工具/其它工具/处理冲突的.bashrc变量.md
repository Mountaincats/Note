要解决两个项目环境变量配置的冲突，核心思路是**避免在 `.bashrc` 中写死路径，而是根据你的需求动态设置**。你有两个项目，不能同时使用两套配置，但可以轻松切换。

### 解决方案（二选一）

**方法一：使用条件判断（推荐，清晰灵活）**
在 `~/.bashrc` 文件中，用 `if` 语句根据一个“开关”变量来决定为哪个项目设置环境变量。

1.  **备份你的 `~/.bashrc` 文件**。
2.  打开 `~/.bashrc`，找到或添加环境变量设置的部分，修改为类似以下内容：

```bash
# ============== YSYX 项目快速切换函数 ==============
ysyx() {
    if [ "$1" = "work" ] || [ "$1" = "red" ]; then
        # 修改 YSYX_PROJECT 的值
        sed -i "s/^export YSYX_PROJECT=.*/export YSYX_PROJECT=\"$1\"/" ~/.bashrc
        source ~/.bashrc
        echo "切换完成！当前项目: $1"
    else
        echo "用法: ysyx [work|red]"
        echo "  ysyx work  切换到 ysyx-workbench 项目"
        echo "  ysyx red   切换到集创赛（redbean）项目"
        echo "当前项目: $YSYX_PROJECT"
    fi
}
# ============================================

# 设置一个“项目开关”，默认为 'workbench'，想用集创赛项目时改为 'red'
export YSYX_PROJECT="work" 
# export YSYX_PROJECT="red" 

# 根据开关动态设置环境变量
if [ "$YSYX_PROJECT" = "work" ]; then
    # 原学习项目的路径
    export NEMU_HOME=/home/mountaincat/ysyx-redbean/nemu
    export AM_HOME=/home/mountaincat/ysyx-workbench/abstract-machine
    export NPC_HOME=/home/mountaincat/ysyx-workbench/npc
    export NVBOARD_HOME=/home/mountaincat/ysyx-workbench/nvboard
    echo "环境变量已设置为: ysyx-workbench 项目"
elif [ "$YSYX_PROJECT" = "red" ]; then
    # 集创赛项目的路径（请根据实际克隆的目录修改）
    export NEMU_HOME=/home/mountaincat/ysyx-redbean/nemu
    export AM_HOME=/home/mountaincat/ysyx-redbean/abstract-machine
    export NPC_HOME=/home/mountaincat/ysyx-redbean/npc
    export NVBOARD_HOME=/home/mountaincat/ysyx-redbean/nvboard
    echo "环境变量已设置为: 集创赛项目"
fi

```

3.  保存文件，然后执行 `source ~/.bashrc` 使配置生效。
4.  **切换项目时**，只需用`ysyx red/work`修改 `YSYX_PROJECT` 变量的值即可。

**方法二：使用别名命令**
创建两个别名命令，分别用于快速切换环境。

在 `~/.bashrc` 中添加：
```bash
alias setup_workbench='export NEMU_HOME=/home/mountaincat/ysyx-redbean/nemu; export AM_HOME=/home/mountaincat/ysyx-workbench/abstract-machine; export NPC_HOME=/home/mountaincat/ysyx-workbench/npc; export NVBOARD_HOME=/home/mountaincat/ysyx-workbench/nvboard; echo “切换到workbench环境”'
alias setup_jichuang='export NEMU_HOME=/home/mountaincat/ysyx-redbean/nemu; export AM_HOME=/home/mountaincat/ysyx-redbean/abstract-machine; export NPC_HOME=/home/mountaincat/ysyx-redbean/npc; export NVBOARD_HOME=/home/mountaincat/ysyx-redbean/nvboard; echo “切换到集创赛环境”'
```
保存后 `source ~/.bashrc`，之后在终端里直接输入 `setup_workbench` 或 `setup_jichuang` 即可瞬间切换。

### 两种方法对比

| 特性 | **方法一（条件判断）** | **方法二（别名）** |
| :--- | :--- | :--- |
| **配置清晰度** | 高，所有配置集中在 `.bashrc` 中，一目了然。 | 中，命令较短，但路径分散在别名定义里。 |
| **切换便利性** | 高，修改一个变量后新开终端即可。 | 极高，在同一个终端内输入一个命令即可。 |
| **持久性** | 切换后在新终端中依然有效。 | 仅对当前终端会话有效，新开终端需要重新执行命令。 |
| **推荐场景** | 需要长时间专注于某一个项目时。 | 需要频繁在两个项目间来回切换时。 |

**总结**：推荐使用**方法一**，它逻辑更清晰，设置一次后打开终端就是想要的环境。如果你需要非常频繁地切换，可以两种方法结合使用。务必根据你克隆的实际目录路径修改上面的示例代码。
