Docker 是一个开源的容器化平台，和传统虚拟机有本质区别

---

### **1. 核心功能**
- **容器化技术**：将应用及其依赖（库、环境变量、配置文件等）打包成一个轻量级、可移植的“容器”，确保应用在任何环境中运行结果一致。
- **解决“在我机器上能跑”问题**：避免因环境差异导致的应用运行问题，实现“一次构建，到处运行”。

---

### **2. 与虚拟机的区别**
| **特性**       | **Docker 容器**                     | **虚拟机**                     |
|----------------|-------------------------------------|-------------------------------|
| **虚拟化对象** | 仅虚拟化应用层，共享宿主机系统内核  | 虚拟完整操作系统+硬件          |
| **启动速度**   | 秒级启动                            | 分钟级启动                    |
| **资源占用**   | 极低，仅存储应用所需内容            | 较高，需分配独立系统资源       |
| **性能**       | 接近原生                            | 有少量虚拟化开销              |

---

### **3. 关键概念**
- **镜像**：只读模板，用于创建容器（比如一个包含 Ubuntu+Python+代码的镜像）。
- **容器**：镜像的运行实例，可独立启动/停止。
- **Dockerfile**：文本文件，定义如何构建镜像。
- **Docker Hub**：公共镜像仓库，可下载现成镜像（如 Nginx、MySQL）。

---

### **4. 常见应用场景**
- **快速部署微服务**：每个服务独立容器，易于扩展和管理。
- **持续集成/部署**：在测试和生产环境使用相同容器，保证一致性。
- **开发环境标准化**：新成员无需配环境，直接运行容器。
- **隔离应用**：不同服务用不同容器，避免依赖冲突。

---

### **5. 简单示例**
```bash
# 从仓库拉取一个镜像
docker pull nginx

# 运行一个容器
docker run -d -p 80:80 nginx
# 现在访问 http://localhost 就能看到 Nginx 页面
```

---

### **为什么用它？**
- **轻量高效**：一台服务器可运行上百个容器，远超虚拟机数量。
- ** DevOps 必备**：实现从开发到生产的全流程环境标准化。

如果你需要配环境或部署应用，Docker 会比传统虚拟环境（如 Python venv）更彻底地解决依赖问题。
