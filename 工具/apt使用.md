## apt使用

`apt` 是现代 Ubuntu/Debian 系统中最核心的软件包管理工具。以下是按功能分类的常用 `apt` 命令，**日常使用记住前四类就足够了**。

### 一、核心维护命令（最常用）
*   `sudo apt update`
    *   **作用**：刷新软件源列表，获取最新的软件包信息。**在安装或升级任何软件前，都应先运行此命令**。
*   `sudo apt upgrade`
    *   **作用**：升级所有已安装的软件包到最新版本（不处理依赖关系的改变，如删除旧包）。
*   `sudo apt full-upgrade` (或 `sudo apt dist-upgrade`)
    *   **作用**：智能升级系统，会处理依赖关系的变更（如需要，会安装新包或删除旧包）。在发行版大版本升级时尤其重要。

### 二、软件包操作命令
*   `sudo apt install <软件包名>`
    *   **作用**：安装指定的软件包。
    *   **示例**：`sudo apt install vim`
*   `sudo apt remove <软件包名>`
    *   **作用**：卸载软件包，**但保留其配置文件**。
*   `sudo apt purge <软件包名>`
    *   **作用**：彻底卸载软件包，**同时删除其配置文件**。
*   `sudo apt autoremove`
    *   **作用**：自动删除那些为了满足其他软件包依赖而安装，但现在不再需要的“孤儿”软件包。定期运行可以保持系统清洁。

### 三、查询与搜索命令
*   `apt search <关键词>`
    *   **作用**：在软件源中搜索包含该关键词的软件包（无需 `sudo`）。
    *   **示例**：`apt search pdf editor`
*   `apt show <软件包名>`
    *   **作用**：显示某个软件包的详细信息，如版本、依赖、描述等（无需 `sudo`）。
*   `apt list`
    *   `apt list --installed`：列出所有已安装的软件包。
    *   `apt list --upgradable`：列出所有可以升级的软件包。
    *   `apt list <软件包名>`：显示该软件包的具体版本信息。

### 四、实用组合与技巧
1.  **一键更新组合**：`sudo apt update && sudo apt upgrade -y`
    *   `-y` 参数表示自动同意所有提示，适合脚本或确认无误后使用。
2.  **模拟操作**：在任何安装或删除命令后加上 `-s` 或 `--dry-run`，可以模拟执行过程而不真正操作，用于检查。
    *   **示例**：`sudo apt install vim -s`

### 五、进阶命令
*   `sudo apt edit-sources`
    *   **作用**：编辑系统的软件源配置文件 (`/etc/apt/sources.list`)。操作前请备份。
*   `apt policy <软件包名>`
    *   **作用**：显示软件包的安装状态、候选版本以及各个软件源的优先级信息。

### 六、`apt`与`apt-get`区别

`sudo apt install` 和 `sudo apt-get install` 的核心功能完全相同，都是用于安装软件包。它们的区别主要在于用户体验和设计理念，`apt` 是更现代、对用户更友好的前端工具。

**简单来说：日常使用推荐 `sudo apt install`，它更直观方便；在编写脚本或需要绝对稳定的输出时，可以使用 `sudo apt-get install`。**

下面是它们的主要区别：

| 对比项 | `apt` (如 `apt install`) | `apt-get` (如 `apt-get install`) |
| :--- | :--- | :--- |
| **设计目标** | 为**交互式使用**设计，提升终端用户体验。 | 为**稳定性和脚本编写**设计，提供稳定、可解析的输出。 |
| **输出信息** | 更友好，有彩色高亮、进度条，显示可升级的软件包数量。 | 输出更原始、简洁，格式固定，便于其他程序解析。 |
| **进度显示** | 有直观的进度条。 | 无进度条，以百分比文字显示。 |
| **推荐使用场景** | **个人电脑日常管理**，操作更直观。 | **Shell脚本、自动化任务**，或需要兼容老系统时。 |

**总结与建议：**
*   `apt` 是 `apt-get`、`apt-cache` 等命令的智能整合与升级，提供了更符合现代习惯的语法和更好的交互体验。
*   对于绝大多数用户，在终端中直接操作时，**使用 `sudo apt install` 是更好的选择**。
*   `apt-get` 并未被淘汰，它在后台脚本、需要确保长期行为稳定的生产环境中依然重要。许多历史教程和脚本也仍在使用它。
