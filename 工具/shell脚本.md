1. Shebang
    * 例子
        ```bash
        #!/bin/bash           # 使用 Bash 解释器
        #!/usr/bin/python3    # 使用 Python 3
        #!/usr/bin/env node   # 使用 Node.js（通过 env 查找）
        #!/usr/bin/perl       # 使用 Perl
        ```
    * #! 用于识别 后面为指定了要使用的解释器路径
    * 必须位于文件​​第一行，必须在​​前两个字节​​（不能有空格开头）

---

2. 执行脚本
   ```bash
   若脚本里包含Shebang,则
   chmod +x try  (可能要给执行权限)
   直接输入脚本文件绝对路径，如: ./try
   
   若不含Shebang，则手动指定使用的Shebang,如:
   <bash> try 例： sh try
   ```

---

3. Bash 中的字符串通过 ' 和 " 分隔符来定义，但是它们的含义并不相同。以 ' 定义的字符串为原义字符串，其中的变量不会被转义，而 " 定义的字符串会将变量值进行替换。

---

4. 在 bash 中为变量赋值的语法是 foo=bar，访问变量中存储的数值，其语法为 $foo。 需要注意的是，foo = bar （使用空格隔开）是不能正确工作的，因为解释器会调用程序 foo 并将 = 和 bar 作为参数。 总的来说，在 shell 脚本中使用空格会起到分割参数的作用，有时候可能会造成混淆，请务必多加检查。

---

5. 
* &&：如果左边失败，右边不执行（因为已经确定结果为假）
* ||：如果左边成功，右边不执行（因为已经确定结果为真）
