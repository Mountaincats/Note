使用 ccache 通常会增加磁盘空间占用，但它是可控的。以下是详细说明和清理方法：

### 1. ccache 是否会占用更多磁盘空间？
**会的。** ccache 的工作原理就是将编译结果（.o 文件等）缓存到指定目录（默认为 `~/.cache/ccache/`），以便下次编译相同代码时直接复用，从而加快编译速度。随着编译不同版本或项目的代码，缓存目录会逐渐增长。

### 2. 如何管理和清理 ccache 缓存？
您可以通过多种方式查看、限制和清理缓存。

#### **a. 查看当前缓存使用情况**
执行以下命令，查看缓存统计信息，包括缓存目录、实际使用大小、文件数量、命中率等。
```bash
ccache -s
```
或者获取更简洁的空间使用情况：
```bash
ccache -s | grep cache_directory
ccache -s | grep -E "(cache size|files found)"
```

#### **b. 设置缓存大小限制（预防性管理）**
这是最推荐的管理方式。您可以配置 ccache，使其缓存总大小不超过指定值。当缓存达到上限时，ccache 会自动清理最旧的缓存文件。
- 设置最大缓存为 5.0 GB：
```bash
ccache -M 5G
```
- 您可以将此命令写入 `~/.bashrc` 或构建脚本中，使其永久生效。

#### **c. 设置缓存过期时间**
可以配置缓存文件的生存时间（TTL），超过此时间的文件将被视为过期并在需要空间时被清理。
- 设置最大缓存时间为 90 天：
```bash
ccache --set-config=max_age=90d
```

#### **d. 手动清理所有缓存（最彻底）**
如果您需要立即释放磁盘空间，可以清空整个缓存目录。
```bash
ccache -C
```
或者直接删除缓存目录（默认为 `~/.cache/ccache/`）：
```bash
rm -rf ~/.cache/ccache/*
```

#### **e. 手动清理旧缓存，保留最近使用的**
此命令会清除所有缓存统计信息，但**不会立即删除缓存文件**。文件会在后续编译过程中被逐步覆盖。如果想强制物理删除，通常需要结合 `-C` 选项。
一种更主动的清理方式是使用 `ccache -c` 来修剪（trim）缓存，但通常更直接的方法是设置大小限制（`-M`）。

#### **f. 按特定项目清理**
ccache 本身不直接支持按项目清理。但您可以通过为不同项目设置不同的 **CCACHE_DIR** 环境变量来隔离缓存，然后单独清理某个项目的缓存目录。
```bash
# 为项目A设置专用缓存目录
export CCACHE_DIR=/path/to/project_a_ccache
# 编译项目A...
# 清理项目A的缓存
ccache -C
```

### 3. 最佳实践建议
1.  **设置合理上限**：首先使用 `ccache -M 10G`（例如 10GB）为缓存设置一个合理的上限。这是“一劳永逸”的自动管理方法。
2.  **定期检查**：偶尔运行 `ccache -s` 查看缓存使用情况和命中率。如果命中率很低（例如低于 20%），可能意味着您的编译环境变化很大（如频繁切换分支、更改核心配置），缓存效用不高，此时可以考虑清理。
3.  **关键项目隔离**：对于非常重要的长期项目，使用 **CCACHE_DIR** 为其设置独立缓存目录，避免被其他项目缓存污染或误清。
4.  **清理后验证**：执行清理操作后，再次运行 `ccache -s`，确认“cache size”已减小。

总结：**ccache 会增加磁盘占用，但通过 `ccache -M` 设置大小限制是管理它的最佳方式。需要立即腾出空间时，使用 `ccache -C` 一键清理所有缓存。**
