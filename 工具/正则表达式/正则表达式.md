# POSIX正则表达式标准与 `\d+` 简写

## POSIX正则表达式标准

**POSIX正则表达式** 是IEEE POSIX标准定义的正则表达式规范，主要用于Unix/Linux系统工具。它有两种主要风格：

### 1. **基本正则表达式（BRE）**
- 传统Unix工具（如grep、sed、vi）的默认模式
- 元字符：`. * ^ $ [ ]`
- 扩展需要转义：`\+ \? \{ \} \| \( \)`

### 2. **扩展正则表达式（ERE）**
- 更现代的标准，支持更多元字符
- 元字符：`. * + ? { } | ( ) ^ $ [ ]`
- **不支持** `\d`, `\w`, `\s` 等简写

## 支持 `\d+` 的正则表达式标准

### **PCRE（Perl兼容正则表达式）**
这是最广泛支持 `\d+` 的标准：

| 标准/库 | 支持 `\d+`? | 特点 |
|---------|------------|------|
| **Perl** | ✅ | 首创者，`\d` 表示数字，`\D` 表示非数字 |
| **PCRE库** | ✅ | 最广泛使用的正则表达式库 |
| **Python re模块** | ✅ | 默认支持 |
| **JavaScript** | ✅ | ES3+ 支持 |
| **Java java.util.regex** | ✅ | 支持 |
| **.NET Regex** | ✅ | 支持 |
| **PHP preg_*函数** | ✅ | 基于PCRE |
| **Ruby** | ✅ | 支持 |
| **Go regexp** | ✅ | 支持 |

### **POSIX ERE vs PCRE 对比**

```c
// POSIX ERE (你的代码目前使用这个)
{"[0-9]+", TK_NUM}           // 匹配数字
{"[[:digit:]]+", TK_NUM}     // POSIX字符类

// PCRE (支持\d简写，但POSIX不支持)
{"\\d+", TK_NUM}             // 在POSIX中不会匹配任何内容！
```

## 你的代码问题分析

你的代码使用的是 **POSIX ERE**（通过 `REG_EXTENDED` 标志）：
```c
ret = regcomp(&re[i], rules[i].regex, REG_EXTENDED);
```

在POSIX ERE中：
- `\d` 会被解释为普通字符 `d`
- 因此 `\d+` 会匹配 "ddd..." 这样的字符串，而不是数字
- 这就是为什么 "1" 无法匹配的原因

## 解决方案

### 方案1：使用POSIX字符类（推荐）
```c
// 使用POSIX字符类
{"[[:digit:]]+", TK_NUM}     // 匹配数字
{"[[:xdigit:]]+", TK_HEX}    // 匹配十六进制数字
```

### 方案2：使用传统字符集
```c
{"[0-9]+", TK_NUM}           // 匹配十进制数字
{"[0-9a-fA-F]+", TK_HEX}     // 匹配十六进制数字
```

### 方案3：切换到PCRE库（不推荐）
如果你必须使用 `\d+`，需要切换到PCRE库：
```c
#include <pcre.h>

// 使用PCRE编译正则表达式
pcre *re;
const char *error;
int erroffset;
re = pcre_compile(pattern, 0, &error, &erroffset, NULL);
```

## 修正后的规则数组

根据你的需求，我建议这样修改：

```c
static struct rule {
  const char *regex;
  int token_type;
} rules[] = {
  {" +", TK_NOTYPE},                      // 空格
  {"==", TK_EQ},                          // ==
  {"!=", TK_NE},                          // !=
  {"&&", TK_LOGAND},                      // &&
  {"0x[0-9a-fA-F]+", TK_HEX},             // 十六进制数
  {"\\$[a-zA-Z_][a-zA-Z0-9_]*", TK_REG},  // 寄存器名
  {"[0-9]+", TK_NUM},                     // 十进制数 (修正：从\d+改为[0-9]+)
  {"\\+", '+'},                           // +
  {"\\-", '-'},                           // -
  {"\\*", '*'},                           // *
  {"\\/", '/'},                           // /
  {"\\(", '('},                           // (
  {"\\)", ')'},                           // )
};
```

## 总结

- **POSIX ERE** 不支持 `\d`, `\w`, `\s` 等简写
- 使用 `[0-9]` 或 `[[:digit:]]` 代替 `\d`
- 确保正则表达式规则顺序正确（特定规则在前，通用规则在后）
- 如果你的代码是NEMU项目的一部分，通常使用 `[0-9]+` 是最兼容的写法
